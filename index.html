What is TOON?

Token-Oriented Object Notation (TOON) is a compact, line-oriented serialization format designed specifically for LLM interactions. It preserves the JSON data model while cutting token usage by 30-60%.

How It Works

JSON repeats field names for every single entry. TOON declares field names once at the top, then lists values in clean rows. The structure stays explicit, but the clutter disappears.

items[2]{id,name,role}:
  1,Alice,admin
  2,Bob,user

Token Savings

The same data in JSON costs ~30-60% more tokens. For a 100-record product catalog:

  • JSON: ~2,847 tokens
  • TOON: ~1,228 tokens
  • Savings: 1,619 tokens per prompt

Run this 1,000 times daily and you've cut millions of tokens monthly.

Key Benefits

  • Token Cost Reduction — 30-60% lower API bills
  • Performance Improvement — fewer tokens = faster responses
  • Improved Accuracy — structured format reduces ambiguity and hallucinations
  • Lossless — convert between JSON and TOON without losing data
  • Human-Readable — clean rows, no clutter
